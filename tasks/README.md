# Tasks Workspace

Kanban-style task management, organized by domain. **Ideas and plans are domain-only** (not in cms/).

## Structure

```
tasks/
├── cms/                        # Completed/historical CMS items only; no new ideas/plans here
│   ├── ideas/                  # (empty — use domain folders)
│   ├── plans/                  # (empty)
│   ├── in_review/
│   └── completed/              # Done CMS-level tasks
├── lavprishjemmeside.dk/       # Client: lavprishjemmeside.dk
│   ├── ideas/                  # Brainstormed tasks for this domain
│   ├── plans/                  # Planner output for this domain
│   ├── in_review/
│   └── completed/
├── ljdesignstudio.dk/          # Client: ljdesignstudio.dk
│   ├── ideas/
│   ├── plans/
│   ├── in_review/
│   └── completed/
└── README.md
```

## Kanban columns

| Folder | Stage | Description |
|--------|-------|-------------|
| `ideas/` | Ideas | Raw brainstormed tasks awaiting planning |
| `plans/` | Plans | Implementation plan created by Planner |
| `in_review/` | In Review | Under review or testing |
| `completed/` | Completed | Done and archived |

## Domain routing

IAN loads and writes **only domain folders** for ideas and plans:
- **Client channels** → `tasks/{domain}/ideas/` and `tasks/{domain}/plans/` (e.g. lavprishjemmeside.dk, ljdesignstudio.dk)
- **Control channel** → `cms/` holds completed/historical items only; no new ideas or plans in cms/

## File naming

`TASK_{SLUG}.md` — generated by IAN Brainstormer via `slugify()`.

## Rules

- All new tasks start in `{domain}/ideas/`
- Planner moves tasks from `ideas/` → `plans/` automatically
- Keep `docs/` for stable reference documentation only
